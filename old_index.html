How I work

The way I operate

01

Understand the flow first

I ask you to walk me through the process the way it actually happens — from the moment a task starts to when it's done. I'm looking for where things slow down, where information gets lost, and where you're doing work that shouldn't need your time.

02

Simplify before automating

I map the process so we both see the full picture. Then I propose what to cut, simplify, or restructure. Only after that do I decide what tools or automations make sense. My goal isn't to build something impressive — it's to solve the real bottleneck.

03

Build systems you own

A lot of automation setups work, but only the person who built them understands how. I name things clearly, organize workflows logically, and leave documentation so you're never stuck when I'm not around. You own it. I just set it up right.
